    1. Упаковка сервисов в Docker¶

    Создано 5 контейнеров:

    • web_app — основное FastAPI-приложение
    • parser_worker — воркер Celery
    • parser_api — FastAPI-сервис для вызова парсера по HTTP
    • db — PostgreSQL 15
    • redis — брокер сообщений Redis

    docker-compose.yml¶

    services:
  redis:
    image: redis:7
    ports:
      - "6379:6379"

  db:
    image: postgres:15
    ports:
      - "5444:5432"
    healthcheck:
      test: [ "CMD-SHELL", "pg_isready -U postgres -d psychologists_db" ]
      interval: 5s
      timeout: 5s
      retries: 5
    environment:
      POSTGRES_DB: psychologists_db
      POSTGRES_USER: postgres
      POSTGRES_HOST_AUTH_METHOD: trust

  web_app:
    build: ./psychologists_service
    ports:
      - "8005:8000"
    depends_on:
      db:
        condition: service_healthy
    environment:
      DB_HOST: db
      DB_PORT: 5432
      DB_NAME: psychologists_db
      DB_USER: postgres
    env_file:
      - ./psychologists_service/.env
    volumes:
      - ./psychologists_service:/code
    command: bash -c "alembic upgrade head && uvicorn app.main:app --host 0.0.0.0 --port 8000"

  parser_worker:
    build: ./parser
    command: celery -A tasks worker --loglevel=info --max-tasks-per-child=10
    depends_on:
      db:
        condition: service_healthy
      redis:
        condition: service_started
    volumes:
      - ./parser:/parser
    environment:
      DB_HOST: db
      DB_PORT: 5432
      DB_NAME: psychologists_db
      DB_USER: postgres

  parser_api:
    build: ./parser
    command: uvicorn server:app --host 0.0.0.0 --port 8010
    ports:
      - "8010:8010"
    depends_on:
      db:
        condition: service_healthy
      redis:
        condition: service_started
    volumes:
      - ./parser:/parser
    environment:
      DB_HOST: db
      DB_PORT: 5432
      DB_NAME: psychologists_db
      DB_USER: postgres

    2. HTTP-вызов парсера¶

    Реализован отдельный FastAPI-сервис parser_api на порту 8010. Он предоставляет два маршрута:

    POST /parse¶

    Сразу запускает парсинг по URL и сохраняет данные в БД.

    POST /enqueue-parse¶

    Ставит задачу парсинга в очередь Celery.

    Файл: parser/server.py

    3. Вызов парсера через очередь (Celery + Redis)¶

    Файл parser/tasks.py:

    import asyncio

import aiohttp
from celery import Celery

from parser import parse_and_save_async

celery_app = Celery(
    "parser",
    broker="redis://redis:6379/0",
    backend='redis://redis:6379/0',
)

# celery_app.config_from_object("celery_config")

@celery_app.task
def run_parsing_task(req: dict):
    url = req["url"]
    table_name = req["table_name"]

    async def wrapper():
        async with aiohttp.ClientSession() as session:
            await parse_and_save_async(session, url, table_name)

    asyncio.run(wrapper())

    Задачи отправляются из FastAPI через:

    run_parsing_task.delay(req.dict())
