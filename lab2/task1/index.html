    Task1

    Тема: Различия между threading, multiprocessing и async в Python.
    Задача: Сравнить время подсчета суммы чисел от 1 до 10 триллионов (10¹³) тремя способами:

    • Потоки (threading)
    • Процессы (multiprocessing)
    • Асинхронность (async/await)

    Требования: - Каждая программа должна содержать функцию calculate_sum(). - Использовать параллельные вычисления (деление задачи на части). - Замерить время выполнения и сравнить подходы.

    Описание решений¶

    1. Threading (threading_sum.py / multithreading_sum.py)¶

    • Идея: Делим диапазон на 8 частей, запускаем 8 потоков.
    • Каждый поток считает сумму на своём интервале и сохраняет в общий список.
    • После завершения всех потоков — собираем результаты.

    Преимущества:¶

    • Потоки быстрее запускаются и переключаются.
    • Низкие накладные расходы.

    Недостатки:¶

    • Ограничение GIL (Global Interpreter Lock) замедляет вычисления на чистом CPU.
    • Потоки эффективно работают при I/O операциях, а не при тяжелых расчетах.

    2. Multiprocessing (processing_sum.py)¶

    • Идея: Делим диапазон на 8 частей, запускаем 8 процессов.
    • Каждый процесс независимо считает свою сумму.
    • Используем очередь multiprocessing.Queue() для передачи результатов.

    Преимущества:¶

    • Нет ограничений GIL.
    • Процессы действительно параллельны на многоядерных системах.
    • Высокая производительность на CPU-bound задачах.

    Недостатки:¶

    • Больше памяти на процессы.
    • Дороже создавать процессы по сравнению с потоками.

    3. Async/Await (async_sum.py / test.py)¶

    • Идея: Используем asyncio и ThreadPoolExecutor.
    • Каждый подзадача (partial_sum) выполняется в отдельном потоке через loop.run_in_executor.
    • В test.py дополнительно тестировались разные количества задач (4, 8, 12, 16, 24, 32).

    Преимущества:¶

    • Отлично подходит для работы с I/O (сетевыми операциями).
    • Минимальная нагрузка на память.

    Недостатки:¶

    • На чистых вычислениях async даёт небольшую пользу (всё равно используется пул потоков).
    • Async не оптимален для тяжелых CPU-вычислений без прямого асинхронного API.

    Результаты измерений¶

    Метод Время выполнения на 10 миллиардах/1 миллиарде с использованием формулы прогрессии Замечания
    Threading ~18-22 секунд/10 секунд
    Multiprocessing ~16-18 секунд/46-47 ctreyl Самый быстрый
    Async/Await ~22-25 секунд/43-45 секунд Самый стабильный

    Выводы¶

    • Multiprocessing работает быстрее всех на больших числах, потому что реально использует все ядра процессора.
    • Threading показывает хорошие результаты, но ограничен GIL.
    • Async в этом случае работает медленно, потому что async эффективен для сетевых запросов, но не для чистых вычислений.

    Итог: Для чистых тяжёлых вычислений типа суммы от 1 до 10¹³ лучший выбор — multiprocessing.
